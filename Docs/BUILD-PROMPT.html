Architecture & rules

Non-negotiable constraints

Drawn from spec §9 and the project CLAUDE.md. Every one is a hard requirement the build must hold to.

Framework. Next.js 15+ App Router, React 18+, TypeScript strict. Every API route Node runtime — export const runtime = 'nodejs'; no edge.
Database. better-sqlite3 (sync), singleton lib/db.ts. DB at Output/db/hteap-demo.db (override HTEAP_OUTPUT_DIR) — not data/. Gitignored.
Styling. CSS custom properties + CSS Modules. No Tailwind, no UI kit — port the prototype CSS directly.
Charts. Hand-rolled SVG components, no chart library. Reuse prototype viewBoxes, bands, gridlines, and labels exactly.
Fonts & network. Self-hosted via next/font. Zero runtime third-party network calls.
Demo clock. DEMO_TODAY = 2026-06-06 via a demo-clock helper; all dates, ranges, and copy derive from it.
Deterministic seed. No random(), no now(); delete DB + reseed → byte-identical. npm run seed idempotent.
13 personas. Featured sarah; 7 diabetes; 5 obesity. Toggle swaps all data via client refetch — no reload, persists per session.
Responsive at 880px. <880 mobile app, ≥880 desktop dashboard; live on resize (matchMedia + useSyncExternalStore); skeleton until mounted.
Animation. Charts animate on scroll (IntersectionObserver 0.35, once/session); honor prefers-reduced-motion; transform/opacity/stroke only.
Accessibility. WCAG 2.1 AA; Lighthouse a11y ≥95; chart SVG role="img" + aria-label + alt table; status by color and icon.
Simulated integrations. All behind /api/*: sync sleeps a seeded 400–1200ms; consent toggles a flag; demo/reset re-seeds. No real FHIR / OAuth / PHI. Permanent fictional-data banner on every route.

Definition of done

Goals — loop until all ten pass

The run does not stop early. The verify→fix loop keeps going until every goal below is green.

G9

Determinism. Delete DB + reseed → identical state.

G10

Full coverage. Every acceptance criterion FR-1…FR-35 is met.

Workflow orchestration

Six phases — sequential foundation, then fan-out

Each phase runs with the Workflow tool and commits on completion, so progress is recoverable.

0

Ground truth Sequential

Decode/confirm the §15 prototypes to disk; extract the five JSON blocks and the SQL DDL into working files. Produce the master checklist — FR-1…FR-35 × ~30 endpoints × ~39 tables — to drive later fan-out.

1

Scaffold Sequential

Next.js app (TS strict), better-sqlite3, next/font, npm scripts, lib/db.ts singleton, demo-clock helper, Output/ dirs, tokens → CSS custom properties, app shell (disclaimer banner, persona context, 880px switch with skeleton-until-mounted).

2

Data layer Sequential · gates all

Schema from #sql-ddl; deterministic, idempotent seed from #spec-seed-data; assert reseed determinism by hashing before/after.

3

API Fan-out

One agent per endpoint group — overview, labs, meds, sync + record locator, gamification, devices, threads, assistant, services, recipes, identity, documents, insights, share, audit, journey, demo-reset. Each reads SQLite and matches #spec-api shapes exactly. Pipeline: implement → adversarially verify shape vs spec.

4

UI Fan-out

Per-route components plus hand-rolled SVG charts (line / bar / gauge) with scroll animation and reduced-motion; mobile patient app and desktop dashboard; copy verbatim. Pipeline: build → verify visual parity against the matching prototype via browser MCP.

5

Verify & fix Loop until green

A verification workflow checks G1–G10 — build, seed determinism, browser-MCP visual parity + persona toggle + 880px switch, Lighthouse a11y, FR acceptance. Collect failures → fan out fixes → re-run. Repeat until all green; log anything deferred with a reason.

How it runs

Operating rules & resume

Fully hands-off: the run resolves ambiguity via the spec's conflict rule, never pauses to ask, and is safe to stop and restart.

Execution

  • Workflow for phase 3–5 fan-out and the verify→fix loop; sub-agents for independent file groups; worktree isolation only when agents would write-conflict.
  • Verification via browser MCP (navigate, resize, screenshot, snapshot) against localhost, plus Lighthouse for a11y.
  • Reuse the CMS design-system tokens/components and the next-best-practices, vercel-react-best-practices, and frontend-design skills.
  • Commit after each phase. Never call random() / now() — derive from DEMO_TODAY.
  • Don't ask — resolve via the conflict rule, finish, then print a short report: what was built, G1–G10 status, how to run, anything deferred.

Resuming after a stop

  • On a re-run, don't start over. Read git log / git status and existing files; evaluate G1–G10 against the tree.
  • Treat the highest committed phase as the resume point — skip finished phases, pick up at the first incomplete one.
  • Always re-run the Phase 5 verify→fix loop at the end, regardless.
  • For an interrupted same-session Workflow, resume it with resumeFromRunId rather than re-spawning agents.
  • Never re-seed or re-scaffold work that already passes its goal.

## Authoritative sources (read first; never invent data or paraphrase copy)

## Non-negotiable constraints (SPEC §9 + CLAUDE.md)
- Next.js 15+ App Router, React 18+, TypeScript strict. Every API route Node runtime (`export const runtime = 'nodejs'`); no edge.
- SQLite via `better-sqlite3` (synchronous), singleton `lib/db.ts`. DB file `Output/db/hteap-demo.db` (base dir `Output/`, override `HTEAP_OUTPUT_DIR`) per `#spec-meta` — NOT `data/`. Gitignored.
- Styling: CSS custom properties (tokens) + CSS Modules. No Tailwind, no UI kit — port prototype CSS directly.
- Charts: hand-rolled SVG React components, no chart library. Reuse prototype viewBoxes / bands / gridlines / labels exactly.
- Fonts self-hosted via `next/font`. Zero runtime third-party network calls.
- Fixed demo clock `DEMO_TODAY = 2026-06-06` via a `demo-clock` helper; all dates/ranges/copy derive from it.
- Deterministic seed: no `random()`, no `now()`; delete DB + reseed → byte-identical. `npm run seed` idempotent. Scripts: dev / build / start / seed.
- 13 personas (featured `sarah`; diabetes `maria robert jim priya hector linda deshawn`; obesity `samuel aisha carol miguel emily`). Persona toggle swaps ALL data via client refetch, no reload, persists per session.
- Responsive at 880px: `<880` mobile patient app, `≥880` desktop dashboard; live on resize (`matchMedia('(min-width:880px)')` + `useSyncExternalStore`); neutral skeleton until mounted (no hydration mismatch).
- Animate charts on scroll-into-view (`IntersectionObserver`, threshold 0.35, once/session); always honor `prefers-reduced-motion: reduce` (final state immediately). Animate transform/opacity/stroke only — never shift layout.
- A11y: WCAG 2.1 AA, Lighthouse a11y ≥95 both views; every chart SVG `role="img"` + descriptive `aria-label` + alt data table; status shown by color AND icon, never color alone.
- Permanent "fictional data / not an official CMS product" disclaimer on every route.
- All "integrations" simulated behind `/api/*`: `sync` sleeps a seeded 400–1200ms then bumps `last_sync_at`; `consent/grant`+`consent/revoke` toggle a flag driving disconnected empty states; `demo/reset` re-runs the seed. No real FHIR / OAuth / identity / PHI.

First read `git log --oneline`, `git status`, and check for `package.json` + which `app/`, `lib/`, and `/api/*` files already exist, then evaluate goals G1–G10 against the current tree. Treat the highest completed phase (per the per-phase commits) as the resume point: skip finished phases, pick up at the first incomplete one, and re-run the Phase 5 verify→fix loop at the end regardless. If a same-session Workflow run was interrupted, resume it with `resumeFromRunId` rather than re-running its agents. Never re-seed or re-scaffold work that already passes its goal.
